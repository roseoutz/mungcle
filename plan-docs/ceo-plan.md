## Vision

### 핵심 전환
원안(service-concept.md)의 "실시간 위치 기반 발견"에서 **"신뢰/안전 중심"**으로 전환.
독립 AI 리뷰어가 "지도의 점은 공원에서 직접 보는 것보다 정보가 적다. 발견이 아니라 접근 장벽이 진짜 문제"라고 도전. 창업자가 수용.

### 10x Check
반려동물의 "신뢰 프로필"이 표준이 되는 세상. 모든 동네 공원에서 앱을 열면 주변 개들의 성향, 안전성, 평판을 볼 수 있고, "같이 걸어도 좋아요" 한 번으로 자연스럽게 합류. 매칭/채팅 없이 반복 관계가 형성되고, 동네 반려동물 커뮤니티가 자동으로 만들어진다.

## 검증된 전제

1. "반려동물 보호자들은 산책 친구를 원하지만, 기존 매칭/채팅 방식의 진입장벽이 너무 높다"
2. "반려동물의 신뢰 정보(성향, 안전성)가 핵심 가치이며, 위치는 이 신뢰를 기반으로 안전한 만남을 지원하는 보조 수단이다" (Second opinion 도전으로 수정)
3. "이 서비스는 사용자 밀도가 충분해야 가치가 있다"

## 기술 스택 (Eng Review에서 확정, 2026-04-10 MSA 전환)

Expo (React Native) + Kotlin/Spring Boot 3.5 MSA + JPA + gRPC + Kafka + Supabase (DB+Storage) + FCM
- 원안 웹 → 앱으로 변경 (푸시 알림 필수)
- 원안 NestJS 모놀리스 → Kotlin MSA 전환 (아키텍처 학습 목적, ADR-010)
- 상세: [eng-review.md](eng-review.md), `ai/decisions.md` (ADR-010~022) 참고

## Outside Voice Findings

독립 AI가 10개 문제를 지적. 3개 적용:

1. **메시징 없음** → Phase 1에 "상호 인사 후 간단 메시지" 추가
2. **9개는 MVP가 아니다** → 2단계로 분리 (Phase 1: 5개, Phase 2: 4개)
3. **산책 예고가 프라이버시 충돌** → Phase 2 + 산책 친구에게만 공개

나머지 7개 (사회성 자기신고, 예방접종 검증 불가, 밀도 수학, Go/No-go 측정 등)는 Known Risks로 기록.

## Known Risks

- 사회성 등급이 자기 신고 기반 → Phase 2 상호 평가로 보완
- 예방접종 사진은 검증 불가 → "인증됨" 대신 "등록됨"으로 표현
- 밀도 문제: 시간대 패턴도 데이터 축적 필요 → 초기 수동 시딩 (한 공원 15-20명)
- Go/No-go 측정: Phase 1에서는 인사하기 전환율로 대체
- 카카오 OAuth 심사 기간 → 이메일 fallback 제공

## Success Criteria

- **핵심 KPI:** "인사하기" 후 상호 인사 전환율
- **보조 KPI:** 개 프로필 등록 완료율, 산책 상태 활성화 빈도, 주간 재사용률
- **Go/No-go:** 첫 2주간 한 동네에서 인사하기 전환율 유의미하게 발생
- **Phase 2 진행 조건:** Go/No-go 통과 AND WAU 100명 이상

## Explicitly NOT in scope

- 정확한 실시간 위치/지도 (200m 그리드만)
- 풀 채팅/DM (간단 일회성 메시지만)
- 배지/포인트/게이미피케이션
- 피드/타임라인 (SNS화 방지)
- 커머스/병원추천
- 사진 갤러리
- 웹 버전 (앱 전용)
- 앱스토어 정식 배포 (TestFlight/내부 테스트로 시작)
