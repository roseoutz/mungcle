### 스택 변경 이력
- 원안: React SPA (웹) → **Expo (React Native) 앱**으로 변경
  - 이유: 푸시 알림 필수. 웹에서는 앱을 안 열면 인사 알림을 못 받음.
  - React/Vite 프론트엔드 코드는 scaffolding만 있어 폐기 비용 0.
- 원안: PWA → **네이티브 앱**으로 변경
  - 이유: iOS PWA 푸시 제약, GPS 백그라운드 제한

## 핵심 기술 결정

### 200m Grid Snap
- GPS 좌표를 200m 그리드 중심점으로 스냅
- 정확한 좌표는 저장/전송하지 않음 (프라이버시)
- 구현: `floor(lat / 0.002) * 0.002` (간단한 수학, PostGIS 불필요)
- GPS 거부 시: 드롭다운으로 수동 동네/공원 선택

### 인사하기 플로우
```
A가 인사 탭 → 서버 저장 → B에게 FCM 푸시
  → B가 "나도 반가워요" 탭 → 상호 인사 완료
  → 양쪽에 대략적 위치 30분 공유
  → 간단 메시지 교환 가능 ("파란 벤치 앞에 있어요")
  → 5분 무응답 시 자동 만료
```

### 알림 시스템
- FCM (Firebase Cloud Messaging) via expo-notifications
- 인사 수신, 상호 인사 완료, 위치 공유 만료 알림
- 인앱 알림 목록 (알림 탭)

### 주요 인덱스
- `walks (grid_cell, status, updated_at)` — nearby 쿼리 성능
- `greetings (sender_id, receiver_id)` — 중복 방지 unique
- `blocks (blocker_id, blocked_id)` — 필터링 성능

## Error Handling

| Codepath | Failure Mode | Handling | User Sees |
|----------|-------------|----------|-----------|
| 카카오 OAuth | API 장애 | 이메일 fallback | "카카오 로그인 실패, 이메일로 진행" |
| 사진 업로드 | Supabase 장애 | retry + 메시지 | "사진 업로드 실패, 다시 시도" |
| GPS 위치 | 권한 거부 | 수동 동네 선택 | 동네 선택 드롭다운 |
| nearby 쿼리 | DB 타임아웃 | 500 + retry | "잠시 후 다시 시도" |
| 인사 푸시 | FCM 전송 실패 | 인앱 알림 fallback | 앱 열면 확인 |
| grid snap | 경계값 | 순수 계산, N/A | 잘못된 cell 배정 (테스트로 방지) |
| 자동 만료 | CRON 미실행 | health check | 산책 상태 안 꺼짐 |

## 성능 고려사항

- **N+1 방지:** `prisma.walk.findMany({ include: { dog: true, user: true } })`
- **CRON:** 1분 간격 auto-expire. NestJS `@Interval(60000)`
- **시간대 패턴:** 초기 50명 규모에서 직접 쿼리 OK. 데이터 쌓이면 사전 집계 테이블
- **동시 접속:** MVP 규모 (50-200명) 성능 이슈 없음
