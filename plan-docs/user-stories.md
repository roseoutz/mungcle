### 핵심 유저 저니 (Happy Path)

```
가입 → 개 등록 → 동네 설정 → 홈
  → 산책 시작("같이 걸어도 좋아요")
    → 주변 개 카드 탐색
      → "인사하기" 탭
        → 상대가 "나도 반가워요"
          → 간단 메시지 + 위치 30분 공유
            → 실제 만남!
```

## 화면별 User Stories

### S0. 스플래시

> 앱을 처음 열었을 때 브랜드를 인식하고, 자동 로그인 여부가 빠르게 판단된다.

| ID | User Story | 수락 기준 | API |
|----|-----------|----------|-----|
| US-000 | 사용자로서, 앱을 열면 멍클 로고를 보고 싶다 | 로고 + 로딩 인디케이터 1.5초 이내 표시 | - |
| US-001 | 재방문 사용자로서, 저장된 토큰이 있으면 자동으로 홈으로 이동하고 싶다 | JWT 유효 → 홈, 만료 → 로그인 화면 | `ValidateToken` |

---

### S1. 로그인/가입

> 최소한의 입력으로 빠르게 가입하고, 바로 개를 등록할 수 있다.

| ID | User Story | 수락 기준 | API |
|----|-----------|----------|-----|
| US-010 | 사용자로서, 카카오 계정으로 한 번에 로그인/가입하고 싶다 | 카카오 버튼 탭 → OAuth → JWT 발급 → 다음 화면 | `POST /api/auth/kakao` |
| US-011 | 사용자로서, 이메일로도 가입할 수 있어야 한다 (카카오 심사 전 fallback) | 이메일+비밀번호+닉네임 입력 → 가입 완료 | `POST /api/auth/email/register` |
| US-012 | 기존 사용자로서, 이메일로 로그인하고 싶다 | 이메일+비밀번호 → JWT 발급 | `POST /api/auth/email/login` |
| US-013 | 사용자로서, 잘못된 비밀번호를 입력하면 명확한 에러를 보고 싶다 | "이메일 또는 비밀번호가 올바르지 않습니다" 표시 | 401 응답 |
| US-014 | 사용자로서, 이미 가입된 이메일로 재가입 시도하면 안내를 받고 싶다 | "이미 사용 중인 이메일입니다" → 로그인 유도 | 409 응답 |

**화면 상태:**
- Default: 카카오 로그인 버튼 + 이메일 로그인/가입 링크
- Loading: 버튼 비활성 + 스피너
- Error: 인라인 에러 메시지 (빨간색)

### S2. 개 등록 (온보딩)

> 내 반려견의 신뢰 정보를 등록하여, 다른 보호자가 안심하고 다가올 수 있게 한다.

| ID | User Story | 수락 기준 | API |
|----|-----------|----------|-----|
| US-020 | 보호자로서, 개의 기본 정보(이름, 견종, 크기)를 등록하고 싶다 | 필수 입력 완료 → 등록 성공 | `POST /api/dogs` |
| US-021 | 보호자로서, 개의 사진을 업로드하고 싶다 | 갤러리/카메라 → 업로드 → 프로필에 표시 | `POST /api/uploads/sign` → Supabase PUT |
| US-022 | 보호자로서, 개의 성향 태그(활발/조용/주의 등)를 선택하고 싶다 | 1~3개 태그 선택 가능 | `POST /api/dogs` (tags 필드) |
| US-023 | 보호자로서, 개의 사회성 등급(1-5)을 입력하고 싶다 | 슬라이더 또는 선택 UI, 기본값 3 | `POST /api/dogs` (socialScore) |
| US-024 | 보호자로서, 예방접종 사진을 등록하여 신뢰를 높이고 싶다 | 사진 업로드 → "등록됨" 뱃지 표시 (선택) | `POST /api/uploads/sign` |
| US-025 | 보호자로서, 사진 없이도 등록할 수 있어야 한다 | 사진 미등록 → 견종별 실루엣 아바타 표시 | - |

**화면 상태:**
- Loading: 사진 업로드 프로그레스 바
- Error: 업로드 실패 → 재시도 버튼
- Success: "등록 완료!" + 카드 미리보기

### S3. 동네 설정 (온보딩)

> 내 활동 반경을 설정하여, 관련 있는 주변 정보만 받는다.

| ID | User Story | 수락 기준 | API |
|----|-----------|----------|-----|
| US-030 | 사용자로서, 내 동네를 설정하고 싶다 | 동네명 검색/선택 → 저장 | `PATCH /api/users/me` (neighborhood) |
| US-031 | 사용자로서, GPS로 현재 위치 기반 동네를 자동 추천받고 싶다 | GPS 허용 → 근처 동네 추천 | expo-location |

**화면 상태:**
- Default: 동네 검색 입력 + GPS 자동 설정 버튼
- Error: GPS 실패 → "수동으로 선택해주세요"

### S4. 홈 (탐색) — 메인 탭

> 산책을 시작하고, 주변에 함께 걸을 수 있는 개를 발견한다.

| ID | User Story | 수락 기준 | API |
|----|-----------|----------|-----|
| US-040 | 보호자로서, "같이 걸어도 좋아요" 산책을 시작하고 싶다 | 토글 ON → 개 선택 → 위치 허용 → 산책 시작 | `POST /api/walks/start` (type: TOGETHER) |
| US-041 | 보호자로서, "혼자 산책 중" 상태로 산책을 시작할 수 있다 | 토글 ON + 혼자 모드 선택 → 주변에 노출 안 됨 | `POST /api/walks/start` (type: SOLO) |
| US-042 | 보호자로서, 산책을 수동으로 종료할 수 있다 | 토글 OFF → 산책 종료 | `POST /api/walks/:id/stop` |
| US-043 | 보호자로서, 주변에 "같이 걸어도 좋아요" 상태인 개 목록을 보고 싶다 | 카드 리스트: 사진 + 이름 + 견종 + 성향칩 + 사회성 색상 + 거리 | `GET /api/walks/nearby` |
| US-044 | 보호자로서, 주변 개 카드에서 "인사하기"를 탭하고 싶다 | 탭 → 내 개 카드와 함께 상대에게 푸시 알림 | `POST /api/greetings` |
| US-045 | 보호자로서, 현재 아무도 산책 중이 아니면 "이 시간대에 자주 산책하는 개"를 보고 싶다 | 빈 화면 대신 시간대 패턴 카드 표시 | `GET /api/walk-patterns/nearby` |
| US-046 | 보호자로서, 60분이 지나면 산책이 자동 종료되길 원한다 | 60분 후 자동 OFF + "산책이 끝났어요" 알림 | 서버 CRON |
| US-047 | 보호자로서, 차단한 사용자의 개는 목록에서 안 보여야 한다 | 차단 유저 양방향 필터링 | 서버 차단 필터 |

**화면 상태:**
- Loading: 스켈레톤 카드 3장
- Empty: 시간대 패턴 + "산책을 시작해보세요" CTA
- Error: "동네를 선택해주세요" 또는 네트워크 에러 + 재시도
- Success: 주변 개 카드 리스트

**산책 토글 상태 다이어그램:**
```
[OFF] ──(토글 ON)──→ [개 선택] ──→ [모드 선택] ──→ [ACTIVE]
                                    |  같이        |  혼자
                                    v              v
                                 [TOGETHER]     [SOLO]
                                    |              |
                        ──(토글 OFF/60분)──→ [ENDED]
```

### S6. 인사 후 메시지 + 위치 공유

> 상호 인사가 완료되면, 간단한 메시지와 대략적 위치를 공유하여 자연스럽게 합류한다.

| ID | User Story | 수락 기준 | API |
|----|-----------|----------|-----|
| US-060 | 보호자로서, 상호 인사 후 간단한 메시지를 보내고 싶다 | 텍스트 입력 (1~140자) → 전송 | `POST /api/greetings/:id/messages` |
| US-061 | 보호자로서, 상호 인사 후 상대의 대략적 위치를 보고 싶다 | 200m 그리드 수준 위치 표시 (30분간) | `GetWalkGridCell` |
| US-062 | 보호자로서, 30분 후 위치 공유/메시지가 자동 만료되어야 한다 | 30분 경과 → "연결 시간이 끝났어요" | 서버 CRON |

**메시지 상태:**
- Loading: "전송 중..."
- Error: "연결 시간이 끝났어요" (만료)
- Success: 메시지 말풍선

### S7. 내 개 탭

> 등록된 내 반려견을 관리하고, 새로운 개를 추가한다.

| ID | User Story | 수락 기준 | API |
|----|-----------|----------|-----|
| US-070 | 보호자로서, 등록된 내 개 목록을 보고 싶다 | 카드 리스트 (사진, 이름, 견종) | `GET /api/dogs` |
| US-071 | 보호자로서, 새로운 개를 추가하고 싶다 | "+" 버튼 → 등록 화면 (S2와 동일) | `POST /api/dogs` |
| US-072 | 보호자로서, 기존 개의 정보를 수정하고 싶다 | 카드 탭 → 수정 화면 | `PATCH /api/dogs/:id` |
| US-073 | 보호자로서, 등록된 개를 삭제하고 싶다 | 삭제 확인 → 삭제 완료 | `DELETE /api/dogs/:id` |
| US-074 | 보호자로서, 최대 5마리까지만 등록할 수 있다 | 5마리 도달 → "+" 버튼 비활성 + 안내 | 서버 검증 |
| US-075 | 보호자로서, 산책 중인 개는 삭제할 수 없어야 한다 | 삭제 시도 → "산책 중에는 삭제할 수 없습니다" | 서버 `FAILED_PRECONDITION` |

### S8. 알림 탭

> 받은 인사를 확인하고 응답하여, 산책 합류로 이어진다.

| ID | User Story | 수락 기준 | API |
|----|-----------|----------|-----|
| US-080 | 보호자로서, 알림 목록을 보고 싶다 | 타입별 알림: 인사 수신, 상호 인사, 메시지, 산책 만료 | `GET /api/notifications` |
| US-081 | 보호자로서, 인사를 받으면 푸시 알림을 받고 싶다 | 앱 백그라운드 → FCM 푸시 + 인앱 알림 | FCM |
| US-082 | 보호자로서, 받은 인사에 "나도 반가워요"로 응답하고 싶다 | 알림 탭 → 인사 카드 → "반가워요" 탭 → 상호 인사 성립 | `POST /api/greetings/:id/respond` (ACCEPT) |
| US-083 | 보호자로서, 받은 인사를 무시할 수 있다 | 무응답 → 5분 후 자동 만료 | 서버 CRON |
| US-084 | 보호자로서, 알림을 읽음 처리하고 싶다 | 개별 읽음 / 모두 읽음 | `POST /api/notifications/:id/read`, `/read-all` |
| US-085 | 보호자로서, 새 알림이 있으면 탭바에 뱃지가 보여야 한다 | 안 읽은 알림 존재 → 빨간 도트 | 로컬 상태 |

**인사 상태 전이:**
```
[인사 수신] ──(반가워요)──→ [상호 인사 성립] ──→ [메시지+위치 공유 30분]
     |                                              |
     └──(무응답 5분)──→ [만료]            (30분)──→ [종료]
```
